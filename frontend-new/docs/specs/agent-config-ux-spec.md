# 代理配置页交互与视觉规范 (UX Spec)

## 1. 导航与页面结构
- **全局导航**: 
  - 在左侧全局导航菜单的“技能库 (Skills)”下方新增“代理 (Agents)”导航入口。
  - **选中态**: 当用户位于代理配置页时，该导航项需呈现选中状态（背景色高亮、主色调文字，与当前系统选中规范保持一致）。
- **页面风格**: 
  - 采用深色运行时控制台 (Dark Console) 风格。
  - **背景色**: 暗色背景 (如 `--bg: #0f1014`)。
  - **面板/卡片背景**: 略浅的暗色底色 (如 `--panel: #15161b`)。
  - **边框/分割线**: 微弱的暗色边框 (如 `--line: #2a2d35`)。

## 2. 页面布局
页面分为左右两个主要区域，采用典型的 Master-Detail（主列表-详情展示）响应式结构。

### 2.1 左侧侧边栏 (机器/代理搜索筛选)
- **搜索框**: 位于顶部，支持按机器名称或代理名称进行文本搜索，带清除 (X) 按钮。
- **筛选标签 (Tabs/Pills)**: 位于搜索框下方。
  - 选项: 全部 (All) / 可用 (Available) / 异常 (Error) / 未安装 (Not Installed)
  - 选中态应有明显的视觉区分（主色调背景或高亮边框）。
- **机器/代理列表**:
  - 默认展示“本机 (Localhost)”分组。
  - 单个机器卡片包含:
    - 机器名称及图标 (区分本机与远程)。
    - 统计信息: 支持的代理总数、当前在线 (Online) 和异常数量，以彩色徽标 (Badges) 或小圆点形式展示状态。
    - 工作负载信息 (Workload): 显示该机器当前的 CPU、内存占用率或运行中的任务数，使用微缩进度条或暗色文本（如 `--muted` 色值）展示，以便直观评估机器负载情况。
    - 选中卡片后，右侧面板变更为该机器的详细代理信息。

### 2.2 右侧主面板 (本机状态与代理列表)
- **页面头部 (Header)**:
  - 显示当前选中的机器名称 (例: Localhost / MacBook Pro)。
  - 显示机器状态徽标 (例: `🟢 在线` / `🔴 离线`)。
  - 辅助信息 (次要文本颜色 `--muted`): 包含操作系统标识、版本号、后台服务 (Daemon) 运行状态及整体负载摘要。
- **全局操作区 (Header 右侧)**:
  - 提供全局操作按钮: 刷新发现 (Refresh Discovery)，带有一个刷新/同步图标。

## 3. 代理表格与状态展示
右侧主区域以数据表格 (Table) 或卡片列表的形式展示该机器下的所有支持代理，视觉上应符合控制台数据密集型展示风格。

### 3.1 表格列与字段定义
1. **代理 (Agent)**: 代理图标 + 代理名称 (如 Claude, Gemini)。
2. **状态 (Status)**:
   - 采用带颜色的标签 (Tag/Badge)。
   - `🟢 可用` (Available): 绿色圆点或背景。
   - `🔴 异常` (Error/Warning): 红色/橘色圆点或背景。
   - `⚪ 离线/未安装` (Offline/Not Installed): 灰色/暗色圆点或背景。
3. **健康度 (Health)**:
   - 展示代理的连接延迟 (如 `12ms`) 或可用性评分 (如 `99%`)。
   - 视觉规则: 延迟低/健康度高显示绿色文本；延迟较高显示黄色文本；无响应显示红色或横线 `-`。
4. **发现模型 (Discovered Models)**:
   - 默认展示自动发现的模型列表。
   - 多个模型时可采用逗号分隔，单行文本截断 (Text Truncation)，悬浮 (Tooltip) 展示完整列表，或用包含数量的标签展示。
   - **强调**: 纯展示默认发现的模型来源，在此处不展示手动填写的模型（普通路径下）。
5. **运行模式 (Run Mode)**: 当前配置的运行模式 (如 Local, Cloud, Docker)。
6. **环境变量 (Env Vars)**:
   - 显示已配置的环境变量数量 (如 `2 Configured` 带图标指示)，不直接明文展示值。
7. **操作 (Actions)**:
   - 详情/编辑 (Edit/Details) 按钮: 点击呼出右侧抽屉。
   - 诊断信息 (Diagnostics) 按钮: 用于查看安装路径、运行日志或诊断数据。

### 3.2 刷新发现反馈
- 用户点击“刷新发现”时:
  - 触发整页或特定行的 Loading 状态。
  - 按钮图标开始旋转，文本变为“正在刷新...”。
  - 刷新完成后通过 Toast 提示刷新结果 (成功/失败)，并解除 Loading 状态。

## 4. 详情抽屉 (Details Drawer)
点击某代理的“详情/编辑”操作后，从右侧滑动呼出抽屉。

### 4.1 抽屉头部与表单编辑区
- **抽屉头部**: 代理图标、名称、当前状态。
- **仅限表单编辑的字段**:
  1. **运行模式 (Run Mode)**: 单选框或下拉菜单。
  2. **代理级环境变量 (Environment Variables)**:
     - 键值对 (Key-Value) 编辑列表。
     - **安全/隐私要求**: 密钥/Token 类型的变量值默认脱敏 (显示为 `••••••••`)，提供一个查看眼睛图标用于明文展示。
  3. **高级模型覆盖 (Advanced Model Override)**:
     - 必须放在一个**默认折叠 (Collapsed)** 的折叠面板 (Accordion/Disclosure) 中。
     - 展开后提供输入框，供用户手动配置特定模型名称，覆盖系统自动发现的结果。
     - 附加提示文本 (Muted text): *"仅在自动发现无法识别特定模型时使用。普通场景下请留空。"*
- **明确不在代理配置页展示的字段**:
  - **推理等级 (Reasoning Level)**: 明确排除，UI 上不可见（该配置后续归入成员配置）。

### 4.2 抽屉底部操作
- 取消 (Cancel) 和 保存 (Save) 按钮，保存时有按钮级 Loading 状态。

## 5. 诊断信息展示 (Diagnostics)
- 点击诊断信息操作，可在抽屉中新增一个 Tab，或弹出独立对话框 (Modal)。
- 内容包含: 代理的安装路径 (Install Path)、版本 (Version)、以及一个自动抓取的错误日志/堆栈面板。采用等宽字体 (Monospace) 和深灰底色的代码块 (Code block) 样式呈现日志。

## 6. 空状态、异常状态与加载状态
- **加载状态 (Loading)**:
  - 页面首次加载时，左侧列表和右侧表格应使用骨架屏 (Skeleton loader)。
- **空状态 (Empty State)**:
  - 左侧搜索无结果时: 展示插画/图标和提示词“未找到匹配的机器/代理”。
  - 右侧当前机器无代理时: 展示插画/图标和提示词“该机器下尚未安装/配置任何代理”。
- **异常状态 (Error State)**:
  - 机器状态探测失败（如离线、服务未响应）时，在右侧头部或页面顶部展示内嵌全局警告条 (Alert/Banner) 提示“机器状态获取失败，仅展示缓存数据”。
  - 代理模型探测失败，状态列变为“异常”，同时展示一个小的警告图标，悬浮提示错误详情。

## 7. 移动端收缩与响应式策略
基于深色控制台风格，移动端需做以下自适应：
- **布局转换**: 隐藏左侧栏 (机器列表) 侧边，转为顶部的一个下拉菜单 (Dropdown) 或抽屉触发按钮，用于切换不同机器。
- **表格响应式**:
  - 窄屏幕下，隐藏次要列（如环境变量、运行模式等）。
  - 极窄屏幕转为卡片列表视图 (Card List View)，每个卡片代表一个代理，将状态、健康度紧凑排列。操作按钮（详情、诊断）折叠至卡片底部的横排，或通过右上角省略号 (Ellipsis) 菜单收纳。
- **详情抽屉**: 在移动端右侧抽屉转为底部弹出的 Bottom Sheet 或全屏弹窗 (Full-screen Dialog)。
